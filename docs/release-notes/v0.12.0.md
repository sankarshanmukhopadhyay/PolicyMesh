# v0.12.0 — Reconciliation Artifacts, Drift Automation, and Pages Delivery

This release turns several roadmap threads into operator-usable surfaces rather than vague future-ware.

## Highlights

### Reconciliation artifacts
- `ReconciliationReport` is richer and now captures status, counts, selected head, fork count, and generation time.
- `links policy reconcile` compares local and remote policy update artifacts and writes durable JSON under `artifacts/reconciliation/`.
- Reconciliation output is suitable for audit trails, incident handling, and operator review.

### Drift automation
- `links policy drift` can now write durable JSON drift artifacts under `artifacts/drift/`.
- `scripts/policy_drift_check.py` adds a cron-friendly operator check with predictable exit codes.
- Drift output includes timestamps, hashes, source URL, and status.

### GitHub Pages via GitHub Actions
- Added `.github/workflows/tests.yml` for continuous test execution.
- Added `.github/workflows/pages.yml` for documentation publishing.
- Added `docs/index.md`, `docs/_config.yml`, and `docs/deploy/pages.md` to make Pages enablement straightforward.

### Correctness and compatibility
- Fixed a policy update signature verification bug in multisignature verification paths.
- Restored backwards-compatible `build_update(...)` support for `quorum_metadata`, string `activation_time`, and dict-based `change_summary`.
- Updated public-facing docs and bumped the package version to `v0.12.0`.

## Upgrade notes
- If you rely on older callers, `build_update(...)` continues to accept historical governance metadata shapes.
- To publish docs, set GitHub Pages to use **GitHub Actions** as the source.
- For recurring checks, schedule `python scripts/policy_drift_check.py <url> <village_id>` in cron or a scheduler of choice.
