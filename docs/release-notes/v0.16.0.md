# v0.16.0 — Federation Endpoints and Quorum Operations

## What this release delivers

v0.16.0 moves PolicyMesh from library-level federation support to live operator-visible protocol surfaces. Nodes can now publish a runtime capability manifest over HTTP, expose a live transparency checkpoint endpoint for peer retrieval, and generate durable quorum inspection artifacts for village governance review.

## Highlights

- Added `GET /villages/{village_id}/transparency/checkpoint` for live checkpoint retrieval.
- Added `GET /nodes/capability` for runtime capability discovery.
- Added `links policy quorum-inspect <village_id>` to inspect effective signer thresholds, weights, and role assignments.
- Stabilized `links.sdk` as the public import surface for capability manifests, checkpoint exchange, and the minimal HTTP client.
- Refreshed documentation across the README, roadmap, policy governance guidance, and docs index.

## Why it matters

The prior release made governance authoring materially stronger. This release makes those federation and governance surfaces operationally legible. Peers can now discover capability posture without reading repository internals, operators can fetch a real checkpoint over HTTP instead of relying on static files, and quorum configuration can be reviewed as a durable artifact instead of an implicit policy detail.

## Upgrade notes

- The service version is now `0.16.0`.
- The new capability endpoint derives its manifest from runtime configuration such as `LINKS_NODE_ID`, `LINKS_STORAGE_BACKEND`, `LINKS_RECONCILIATION_MODE`, and `LINKS_NODE_SIGNING_KEY_B64`.
- If a node signing key is configured, the checkpoint endpoint signs the returned checkpoint automatically.
