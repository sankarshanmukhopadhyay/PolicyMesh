# v0.17.0 — Decision Receipts and Alertable Drift

## What this release delivers

v0.17.0 shifts PolicyMesh from operator-visible governance tooling to **evidence-producing governance execution**. Policy pull now emits machine-verifiable decision receipts, lifecycle-aware admission checks distinguish `apply`, `defer`, and `reject`, and the cron-friendly drift checker can trigger automation only when drift status changes.

## Highlights

- Added `schemas/policy_decision_receipt.schema.json` for durable policy decision evidence.
- Added automatic receipt generation under `artifacts/receipts/<village_id>/` during `links policy pull`.
- Added `links policy verify-receipt <path>` to validate receipt integrity and optional signatures.
- Added lifecycle-aware policy update evaluation with explicit defer and reject outcomes.
- Added drift automation support through `--state-file`, `--on-change-cmd`, and `--webhook-url` in `scripts/policy_drift_check.py`.
- Added signed and unsigned example receipts for apply, defer, and reject outcomes.
- Refreshed README, roadmap, Pages index, and operator docs to reflect the new evidence layer.

## Why it matters

Before this release, PolicyMesh could reconcile and apply updates, but the operator had to infer why a specific policy head was accepted, deferred, or blocked. That is workable for a development tool. It is weak for an assurance surface.

This release makes governance execution legible:

- a selected update now produces a durable receipt
- admission outcomes now carry machine-readable reason codes
- reconciliation evidence is linked directly into the receipt
- drift checks can trigger operational responses when state changes, not on every poll

That moves PolicyMesh closer to a release profile that can support audit, federation troubleshooting, and higher-assurance review workflows.

## Upgrade notes

- The service version is now `0.17.0`.
- `links policy pull` now writes a decision receipt even when `--apply` is not used.
- If `LINKS_NODE_SIGNING_KEY_B64` is configured, generated receipts are signed with the node key.
- Drift automation remains optional. Existing cron usage continues to work without changes.
